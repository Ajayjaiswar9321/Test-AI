## 1. PROJECT PURPOSE - Why This Was Built

**Test AI** is an AI-powered automated testing platform that lets anyone test any website without writing a single line of code. You just enter a URL, describe what you want to test, and the AI generates test scenarios, writes Playwright scripts, executes them in a real browser, and gives you a full report.

### Problem It Solves
- Manual QA testing is slow, repetitive, and expensive
- Writing Playwright/Selenium test scripts requires coding knowledge
- Setting up test automation frameworks takes hours/days
- Non-technical team members (PMs, designers) can't run tests themselves

### Who It's For
- QA Engineers who want to speed up test creation
- Developers who want quick smoke tests on any URL
- Product Managers who want to verify features without coding
- Anyone who needs automated browser testing without setup

### What Makes It Different
- **Zero setup** - paste a URL and go
- **AI-powered** - generates test scenarios from page analysis
- **Live preview** - watch tests execute in real-time with screenshots
- **AI failure analysis** - when a test fails, AI suggests what went wrong
- **Chat assistant** - ask questions about Playwright, testing strategies
- **Export** - download generated Playwright scripts & CSV reports

### Key Functions in server.ts
| Function | Purpose |
|----------|---------|
| `analyzePage(url)` | Headless Playwright crawl - extracts headings, buttons, links, inputs, forms, login detection, search detection, nav detection |
| `isOllamaAvailable()` | Checks if Ollama is running at localhost:11434 |
| `ollamaChat(system, user)` | Sends chat to Ollama REST API |
| `reloadGeminiKey(key)` | Hot-reload Gemini API key without restart |
| `LocalAIEngine.generatePlaywrightCode()` | Keyword-based Playwright script generator |
| `LocalAIEngine.generateDynamicScenarios()` | Generates 8-10 test scenarios from page analysis |
| `retrieveContext(query)` | RAG keyword search across knowledge base |

### Data Flow
```
User enters URL + goal
  -> POST /api/ui-plan
    -> analyzePage() crawls URL with headless Playwright
    -> AI generates test scenarios (Gemini > Ollama > Local)
    -> Returns scenarios to frontend

User clicks "Execute All"
  -> POST /api/run-tests (for each scenario)
    -> Playwright launches browser
    -> Executes each step
    -> SSE streams progress to frontend via /api/events/:runId
    -> On step failure: captures screenshot + page text
    -> Asks AI for failure suggestion
    -> Emits "suggestion" SSE event
    -> Frontend shows amber AI Suggestion box

Test completes
  -> Full report generated (steps, pass/fail, duration)
  -> Script saved to workspace/generated-scripts/
  -> CSV export available
```

### Real-time Communication
- **Server-Sent Events (SSE)** via `/api/events/:runId`
- Event types: `step` (progress), `suggestion` (AI help), `script` (generated code), `info`, `success`, `error`
- Frontend `ConsoleStream` component connects via `EventSource`
- Uses refs for callbacks to prevent SSE reconnection on re-render

### Authentication
- JWT-based (jsonwebtoken library)
- Password hashing with bcryptjs
- Token stored in localStorage
- `authenticate` middleware on all protected routes
- Auto-logout on 401 responses

## 4. CHATBOT DETAILS

### How the Chat Works
1. User types a question in `SidebarFixChat` component
2. Frontend sends POST to `/api/chat` with `{ code, message }`
3. Server checks AI priority:
   - **Gemini available?** -> Uses `@google/generative-ai` SDK with `gemini-2.0-flash` model
   - **Ollama available?** -> REST API call to `localhost:11434/api/chat` with `gemma2:2b`
   - **Neither?** -> `LocalAIEngine` keyword matching
4. RAG context injected: `retrieveContext(message)` searches 22 knowledge entries
5. System prompt LOCKS chat to Playwright/testing topics only - rejects other topics
6. Response rendered as Markdown via `react-markdown`
7. If response contains code fix, shows "Apply Fix" button

### Chat System Prompt (simplified)
```
You are Test AI - an expert QA engineer. Only answer questions about:
- Playwright testing
- Test automation
- Debugging test failures
- Test strategies
Reject all other topics.
```

### AI Failure Analysis Flow
When a test step fails during execution:
1. Playwright captures page screenshot (base64)
2. Playwright captures page text content
3. Server sends failure context to AI (Gemini/Ollama/Local)
4. AI analyzes what went wrong and suggests fix
5. Suggestion emitted as SSE event (`type: "suggestion"`)
6. `ConsoleStream` shows amber "AI Suggestion" box
7. Chat sidebar auto-opens if closed

### Libraries Used in Chat
| Library | Purpose |
|---------|---------|
| `@google/generative-ai` | Gemini API SDK |
| `react-markdown` | Render AI responses as formatted markdown |
| `lucide-react` (Bot, User, Sparkles, Wand2, Settings) | Chat UI icons |
| `motion/react` | Message animations (spring, fade) |

7. **HUD elements** - Corner brackets, circuit decorations, status LEDs
